The test data has been evaluated in unit increments from -49.5 to 49.5 in both directions on the the complex plane. A green dot above indicates that the same calculation with this library matches the test data to ten significant digits. A yellow dot indicates a variance of one digit in the significand, i.e. the part of the value in scientific notation without the exponent. A red dot indicates a variance of two digits or more, where again this difference is in the significand. A black dot is either a large error or NaN.

Functions requiring more that one argument, such as Bessel or elliptic functions, indicate the additional parameters in the function name in square brackets. Most of these parameters precede the main argument in the function call. When they do not an empty list is included in the file name to allow detection of these cases.

These tests give an immediate visual sense of the accuracy of this library. They can also be used to identify argument domains that need attention.

Known issues with data from mpmath as of 2022.09.18:

The imaginary parts of error functions are incorrect in large regions of the complex plane. These show as significant black wedges for imaginary erf and erfc, as well as real erfi.
The imaginary part of the zeta function is inaccurate in the far right-hand plane. This shows as a strip of red on the right-hand edge above.
Hankel functions with large imaginary parts are inaccurate in either the upper or lower half plane with standard precision. This can be overcome with a massive increase in precision when generating test data.
